Android Studio 从入门到跑路

前言

Android 开发现在的语言用的都是Kotlin,可惜我们学校教的是JAVA，目前来看有JAVA基础入门Android开发不是很难。本篇博客大部分都是基础笔记，真要到进阶和项目实战我估计我已经跑路了

一 . APP视图下Android Studio 用到的重要的文件

1.manifest目录

在app/manifests目录中，有一个AndroidManifest.xml文件

2.AndroidManifest.xml

AndroidManifest.xml文件是一个应用清单，这个文件包含了APP配置信息，系统需要根据里面的内容运行APP代码，显示页面

详情可以大佬写的位置 AndroidManifest.xml详解

3.Activity

  • Android组件中最重要四大组件之一

  • Activity是一个应用程序组件，提供一个屏幕，用户可以用来交互为了完成某项任务。

  • 一个Activity通常就是一个单独的屏幕

4.layout/activity_main.xml

android的布局文件，在文件中写界面的整体布局

二 . Android视图

1.Activity 的创建

在Android视图下你只需要把鼠标放在manifest目录，右击new选择Activity选择需要创建的Activity视图，在弹出的选项里可以选择关联的XML文件以及所用的编程语言

2.Activity的生命周期

Activity存在四种状态，每种状态对应调用不同的接口方法

  • Running(运行)
  • Paused(暂停)
  • Stopped(停止)
  • Destroyed(销毁)

接口方法：

名称 调用时间
onCreate(Bundle savedInstanceState) 创建 Activity 时调用。 设置在该方法中，还以 Bundle 的形式提供对以前储存的任何状态的访问。其中参数 savedInstanceState 对象是用于保存 Activity 的对象的状态。
onStart() Activity 变为在屏幕上对用户可见时调用。
onResume() Activity 开始与用户交互时调用（无论是启动还是重启一个活动，该方法总是被调用）。
onPause() 当 Android 系统要激活其他 Activity 时，该方法被调用，暂停或收回 CPU 和其他资源时调用。
onStop() Activity 被停止并转为不可见阶段时调用。
onRestart() 重新启动已经停止的 Activity 时调用。
onDestroy() Activity 被完全从系统内存中移除时调用。 该方法被调用可能是因为有人直接调用 finish() 方法或者系统决定停止该活动以释放资源。
3.Activity的跳转

每一个Activity都是一个单独的活动页面，我们要进入下一个Activity就需要页面跳转

跳转所需要用的监听事件是 ‘onClick()’

首先我们创建一个新的Activity命名为SecondActivity

在XML文件中添加一个Button 按钮 然后设置按钮的位置大小，添加跳转事件

        < Button
            android:id="@+id/button3"                
            android:layout_width="match_parent"
            android:layout_height="wrap_content"
            android:text="跳转到第二个Activity"
            android:onClick="SecondActivity"
            />

android:onClick=”SecondActivity” 后面的SecondActivity 是你要跳转的Activity页面名称

在写完第六行代码的时候你会发现SecondActivity报错，因为你没有在SecondActivity.class文件里重写跳转方法

冷知识：快速写跳转方法快捷键 Alt+Enter

在SecondActivity.class文件里写跳转方法

    public void SecondActivity(View view) {
        startActivity(new Intent(this,SecondActivity.class));   
    }
