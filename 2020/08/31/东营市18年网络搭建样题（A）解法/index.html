东营市18年网络搭建赛题（A）解法

这种比赛年年都是主办方拿奖，年年都有黑幕，比赛还有什么意思？

准备工作

所有linux主机都要进行以下操作

1.设置ip

vi /etc/sysconfig/network-scripts/ifcfg-ens33

ROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=none
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
NAME=ens33
UUID=2a5f38d6-c019-4e5d-b602-a37b3f297db1
DEVICE=ens33
ONBOOT=yes
IPADDR=172.168.70.166
PREFIX=24
GATEWAY=172.168.70.161
DNS1=8.8.8.8
PEERDNS=no

2.配置Yum源

懒得写了 看我这篇播客吧 Yum源

3.最最最最重要的一条命令，关闭防火墙和selinux

setenforce 0

systemctl stop firewall

Centos-A3

（一）完成域名服务器部署
1.安装bind服务

yum install -y bind

2.修改配置文件

vim /etc/named.conf

options {
        listen-on port 53 { any; };    #any监听所有端口
        listen-on-v6 port 53 { ::1; };
        directory       "/var/named";
        dump-file       "/var/named/data/cache_dump.db";
        statistics-file "/var/named/data/named_stats.txt";
        memstatistics-file "/var/named/data/named_mem_stats.txt";
        allow-query     { any; };    #any允许所有人查询
        blackhole    { 192.168.70.0/24; };    #禁止此网段访问此DNS服务器
3.添加正反向解析

centos 的正反向解析居然不是添加到全局配置文件，以前按照学Debian的思路来学centos居然踩坑了

vim /etc/named.rfc1912.zones

zone "2018skills.com" IN {
        type forward;
        forward only;
        forwarders { 192.168.10.103;};
};

# 正向解析
zone "lin.2018skills.com" IN{
        type master;
        file "lin.2018skills.com.zone";    #区域解析记录文件默认在 /var/named/,我按照Debian的方法直接写路径会报错
};

# 反向解析
zone "70.168.172.in-addr.arpa" IN {
        type master;
        file "172.168.70.arpa";
};
4.创建正反向解析记录文件

Centos也是提供模板的 只需要复制就可以

cp /var/named/named.localhost /var/named/lin.2018skills.com.zone

cp /var/named/named.loopback /var/named/lin.172.168.70.arpa

vim /var/named/lin.2018skills.com.zone

$TTL 1D
@       IN SOA  lin.2018skills.com.  root.lin.2018skills.com. (
                                        0       ; serial
                                        1D      ; refresh
                                        1H      ; retry
                                        1W      ; expire
                                        3H )    ; minimum
        IN NS dns.lin.2018skills.com.
dns     IN A  192.168.70.166
ns      IN A  192.168.70.166
*       IN A  192.168.70.166

# 格式一定要正确
vim /etc/named/172.168.70.arpa 

$TTL 1D
@       IN SOA  lin.2018skills.com.  root.lin.2018skills.com. (
                                        0       ; serial
                                        1D      ; refresh
                                        1H      ; retry
                                        1W      ; expire
                                        3H )    ; minimum
        IN NS dns.lin.2018skills.com.
166     IN NS dns.lin.2018skills.com.
166     IN PTR dns.lin.2018skills.com.
        IN A   172.168.70.166

检查主配置文件和区域解析文件错误

named-checkconf /etc/named.conf

named-checkzone /var/named/lin.2018skills.com.zone

named-checkzone /var/named/172.168.70.arpa

5.测试

nslookup dns.lin.2018skills.com

nslookup 172.168.70.166

Cenros-A4

（一）完成网站服务器部署

创建网站 http://www.lin.2018skills.com：8080,其网站主目录为 /www/8080 ，主页名为 skills8080.html，首页内容为“Welcome chinaskills’ s website：8080” ，被访问端口为8080，虚拟目录名为 vdir

1.创建网站

mkdir /www/8080

2.修改主页内容

echo "Welcome chinaskills’ s website：8080" > /www/8080/skills8080.html

3.修改配置文件

网上很多教程都是单独创建一个配置文件然后在全局配置文件添加位置，我按照网上教程完成后各种报错报了一上午！！！最后还是决定在全局配置文件修改

vim /etc/httpd/conf/httpd.conf

# 数字是行号

......
42 Listen 8080    #监听端口
......
86 ServerAdmin root@localhost     #管理员邮箱地址
......
95 ServerName www.lin.2018skills.com    #网站服务器域名
96 Alias /vdir "/www/8080"     #虚拟目录
......
119 DocumentRoot "/www/8080"   #网站数据目录
......
124 <Directory "/www/8080">    #网站数据目录权限
125     AllowOverride None
126     # Allow open access:
127     Require all granted
128 </Directory>
......
163 <IfModule dir_module>
164     DirectoryIndex index.html skills8080.html    #默认网站索引页面
165 </IfModule>
......
4.检查配置文件

httpd -t

5.修改hosts

vim /etc/hosts

127.0.0.1   localhost localhost.localdomain localhost4 localhost4.localdomain4
::1         localhost localhost.localdomain localhost6 localhost6.localdomain6
172.168.70.166   www.lin.2018skills.com   ip #网站指向你的网站

6.重启httpd 在浏览器输入你的域名查看网页

systemctl restart httpd
